# Fundamental Functions

## LOOKUP & VLOOKUP & HLOOKUP <Badge type="warning" text="Not Recommended" />

`VLOOKUP` is one of the most commonly used advanced function to retrieve value using a offset.
`VLOOKUP(target, range, col_idx, [fuzzy_match])` searches the target value in the specified range of column, and **use the target index found to retrieve another value from another column range.**
Note that `col_idx` is the **integer index relative to the given search range**, that is why people have to pass 2-dim range(to include the column to retrieve value from) to `VLOOKUP` even though it only search the first column.

`HLOOKUP` works similarly by searching the first row instead. And `LOOKUP` is a more generic version covers both `VLOOKUP` and `HLOOKUP` with more ambiguity of its usage.

All of these `*LOOKUP` functions are **NOT RECOMMENDED**, everyone should be using [Match & INDEX](#match-index-badge-typetip-textprefer-this-over-lookup-family) instead.

> [!IMPORTANT]
> `*LOOKUP` family are considered BAD because it relies on the selected 2-dim range to represent to data source column/row.
> If we insert a new row/column between the original range(Excel isn't smart enough to update the 2-dim range), the whole reference would be broken.

## MATCH & INDEX <Badge type="tip" text="Prefer this over *LOOKUP family" />

`*LOOKUP` family is not intuitive for people with programming background because they mix two steps into one.
And the name *LOOKUP* doesn't imply it would apply the index to another column/row.

A more readable solution is the combination of `MATCH` and `INDEX`.
`MATCH(target, range, [match_type])` returns a index found on `range` by the `target` value.
And we apply this index to `INDEX(range, row_idx, [col_idx])` to retrieve the value from another range.

> Yes `INDEX` can retrieve value from 2-dim range, with two indices.

Of course the index returned by `MATCH` is relative to the search range, would it retrieve a expected value depends on the range taken by `INDEX`.

```vb
=INDEX(A1:A4, MATCH("foo", B1:B4, 0))
```

> [!NOTE]
> `0` in the formula above means *exact match*, see [documentation](https://support.microsoft.com/en-au/office/match-function-e8dffd45-c762-47d6-bf89-533f4a37673a)
