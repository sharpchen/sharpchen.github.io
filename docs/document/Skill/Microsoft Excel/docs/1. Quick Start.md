# Quick Start

## Excel has Implicit Language

The reason why Excel is making people furious is, it has a very limited expressiveness and doing to much things in an implicit way.

If you inherit formula from one cell to another, Excel would increment the column index and row index automatically, if you don't like it you have to use `$` to mark the index as a fixed reference.

| A | B | C |
| ------------- | -------------- | -------------- |
| `=COUNTA(B2:Z$2)` | Item1 | Item2 |
| `=COUNTA(B3:Z2)` | Item1 | Item2 |

### Data Format & Date View

Any cell has a **Regular** format(meaning no format) by default. The cell format would change on manual setting or first data input.
If the target cell has a regular format, Excel would try **parsing the coming value(every input is string)**, if the value is a `2000/1/17` Excel would **implicitly change the cell format** to date and so on.
Once the cell format is determined for the first time, the format can only be changed by manual setting.

Any incoming value to a cell with a non-regular format would be coerced to the target format, if parsing failed Excel would show no error until you use the invalid value to calculate anything.
Excel may show `#VALUE!` on the cell that has invalid calculation result.

The actual user input string would stay in the workbook storage as a data source, the view of the cell simply displays the appearance of the input in certain format.
Cell view and its format are coupled concepts, the format is a type, and the view is its string format. You can select different views for cells of same format.
For example, a value of date type can be `2025-1-10` or `2025/1/10` depending on what you want.

## Excel is Lisp-Like

Excel formulas are in the form of nested function calls. It's kind of lisp except it has its brackets on right.
Excel 2021 introduced [`LET`](https://support.microsoft.com/en-gb/office/let-function-34842dd8-b92b-4d3f-b325-b8b8f9908999) function allowing to declare variables in your one-liner formula.

```vb
; var foo = 1
; return foo + 1
=LET(foo, 1, foo + 1)
```
Excel 2024 introduced [`LAMBDA`](https://support.microsoft.com/en-us/office/lambda-function-bd212d27-1cd1-4321-a34a-ccbf254b8b67) function to define your one-liner function literal.

```vb
; ((a, b, c) => a + b + c)(1, 2, 3)
=LAMBDA(a, b, c, a + b + c)(1, 2, 3)
```

So nowadays we already have a lisp-like language in Excel:

```vb
=LAMBDA(a, b, c,
  LET(
    determinant, b ^ 2 - 4 * a * c,
    realPart, ROUND(-b / (2 * a), 5),
    complexPart, ROUND(SQRT(ABS(determinant)) / (2 * a), 5),
    output, HSTACK(
      VSTACK("X = ", "X = "),
      IF(
        determinant < 0,
        VSTACK(
          COMPLEX(realPart, complexPart),
          COMPLEX(realPart, -complexPart)
        ),
        VSTACK(realPart + complexPart, realPart - complexPart)
      )
    ),
    output
  )
)
```

## Data in Excel is Scattered

Any querying or formula are manipulating on **Structure of Arrays** instead of Array of Structures.
Excel functions expects single cell or a range, instead of groups of actual values.
You're not allow to constructing tuples directly but intermediated by ranges.

Sometimes you might need temporary value storage like variables in imperative programming, the equivalence in Excel is a map of cells.
There's no way to refresh a same cell as variables but a whole linear range(either row or column) of cells, it's in fact a visualization of functional map.
**You allocate the memory footprint in the spreadsheet layout**, you're programming in a visualized way.
